    Fix #524 and #526: Broken in non-English Thunderbird
    
    A string added in the Firefox 60 fix wasn't copied to the non-English `messages.json` files, which meant that it wasn't in the non-English `strings.properties` and `string.dtd` files.     To help mitigate this in the future, to some degree, the `i18n.js` script makes sure that all English strings are present in the other languages when creating the the .properties and .dtd files. But I still have to remember to run that script.

    Fixed bug: Rendering and reverting the same selection would result in loss of Markdown
    
    Pretty major change in the render-revert logic. It now looks for
    wrapper inside the current selection, as well as wrappers above it. So
    it's now more aggressive about reverting, but that's good: a) failing
    to revert when we should results in loss of Markdown, which is
    equivalent to "loss of user data"; b) it probably better represents the
    user's intention. Has the cool side-effect of being able to revert
    multiple chunks with the same big selection.
